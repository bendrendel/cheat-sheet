Sectioning elements

Sectioning elements are a subset of semantic elements including section, article, nav, header, footer, aside, and figure and should be used in place of divs

Before semantic HTML was introduced in HTML5, developers used lots of divs with class names to section up website, semantic html standardizes that better to help machines read your site

Document Outlines

Every HTML document has a heirarchical outline that is defined by the heirarchy of heading tags h1-h6, this plays a significant role in search engines determining what's important on the page and for accessibility. avoid skipping heading levels.

main

main should only be used once per document, representing the main content of the body

Sections

The section element is a way of creating sections in a document, they are generic and not exactly clear how to leverage them properly

The section element will create a top-level spot in the document outline, interrupting the heirarchy of headings, but it shouldn't be used to manipulate the document outline like this, rather the headings should always define the document outline without relying on sections

Sections should always contain at least one heading element h1-h6, or else it will create an "Untitled Section" node in the document outline. With a heading, the heading name will be used in the document outline instead.

Sections should not be used for self-contained content (article should be used), or for purely layout purposes (div should be used in that case)

Navs

Navs are great for search engines to identify all pages of your site, can have multiple navs per page if you have different sets of related links

Headers

The header element is for introductory content, best practice to use at top of website to contain name/logo/navigation. headers associated with nearest parent sectioning element, so can have a header inside an article for instance, meaning you can have multiple headers on a page

Footers

Footers can contain navs, copyright notices, author bios, etc. Like headers, you may have a main site footer, and could have footers within articles

Asides

Aside elements should be used within articles to separate the content from the rest of the article like advertisments, definitions, stats, or quotations. They should also be used outside an article for the webpage as a whole for sidebars that might contain a nav or several navs with links

Div

Divs should still be used for pure layout purposes, often used with flexbox or when you need containers to center things, etc. Don't use semantic sectioning elements in these cases as it will just be confusing

Articles

An article about Articles

Articles define content that makes sense on its own and could be plucked out of the page and distributed in a totally different context like aggregator sites or a search engine

Articles are essentially mini web pages within your HTML document and can have their own headers, sections, footers, etc separate from the rest of the site

A site might have only one article element, or could have many like multiple blog posts on a page, which is like merging multiple HTML files into a single document

Other semantic Elements (Not for sectioning)

The time element allows you to wrap dates and times and include a datetime attribute that helps search engines, etc. interpret it, can just include date or can also include time (24hr) and offset from GMT (-008 represents pacific standard time)

CSS/Legacy Browser

all modern browsers support HTML5 which introduced the semantic elements

But sometimes you will see the following css to set the semantic elements as block for legacy browsers

section, article, aside, footer, header, nav { display: block; }
