Logical Errors

Bad logic that doesn't stop execution like a runtime error does, but it produces incorrect output

  • To find logical errors, starting at the top of your code, insert temporary console.log() after each piece of logic to print all relevant variables until you find a value you didn't expect


Runtime Errors

A thrown error that haults execution of code unless error handling is used, and will display error information in console

Built-in runtime errors

javascript will automatically throw an error if the compiler cannot interpret your code

  • SyntaxError: typos, etc that the compiler can't interpret
  • ReferenceError: trying to use a variable that hasn't been declared
  • TypeError: attempting to perform operations on values of the wrong type
  • +4 other built in JavaScript error types that are less frequently seen

Custom runtime errors

In your code you can create a custom error object and throw it

  • Create an error object: Error('Custom error message.') or new Error('Custom error message.')
  • Throw an error object: throw Error('Custom error message.')

Error Stack Trace

Error information is logged a s an error stack trace, a message that includes the error location (file path/name:line number), type of error, and an error message

Runtime Error Handling

Point is to allow your program to continue running when a runtime error is thrown, instead of default behavior of stopping all execution when an error is thrown

  • use a try...catch statement, as in try { ... } catch (e) { ... }
  • if an error is thrown in the try block (whether a built-in error or a custom thrown error), then the catch block will execute with the thrown error object e, then any code after the try...catch statement will run
