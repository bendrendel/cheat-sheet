Basic Concepts

Affordances: are what something affords you the ability to do, whether or not it was intended by the designer

Signifiers: are what indicate the affordances of something, like a cup handle signifies it can be picked up

Skeuomoprhism: UI elements imitating real-life counterparts, offers familiar signifiers that may help lower learning curve for users

Flat design: UI elements relying on signifiers that don't imitate real-life, more prevelant as digital age continues, typically more simple and less cluttered than skeuomorphism

Primary navigation: Most important links/buttons to be displayed on every page

Secondary (breadcrumb) navigation: Links/buttons displaying the pages or attributes that led to the current page

Link States

Links have four states with associated CSS pseudo-classes, :link, :hover, :active, :visited

CSS rules must be in order :link, :visited, :hover, :active for proper cascading e.g. a link could be both visited and hovered over at the same time, so you want the hover to come second in your css so that it overrides the visited style

hover states are not accessible in mobile browsers

Link styling

Traditionally link and hover states are blue and underlined (less common to underline more recently), active is red, visited is purple

Links in text passages traditionally underlined while navigational links are not

Also consider altering background-color, font-weight, and border

Link style, in every state, should be different than any other text on page

Content of anchor element, the anchor text, should be descriptive of link for usability, accessibility, and SEO

Home Title attribute creates tooltip, often helpful on links especially when link is an image or icon

Cursor property should be set to pointer on hover (default in most browser's user agent stylesheet)

Buttons

Hover state at minimum should change pointer, active state should be obvious

Navigational buttons should always have text to remove any uncertainty about what it does

Secondary (breadcrumb) navigation

Location-based: shows where you are within structure of website, e.g. Shopping > Fashion > Shoes

Attribute-based: shows selected attributes of the page you are on, order doesn't matter, often have an x to remove, e.g. > Flats > Brown

Path-based: uncommon, shows unique path user took to get to page, so breadcrumb trail can vary for each user

Provides context, understanding of scope of site, easy backward navigation, but it would be extraneous for a site that only has a few pages

Typically horizontal list, in header, left-aligned, below primary navigation, separated by ">" or "/", not underlined, when too long use elipses for beginning: ... > About > Register

Breadcrumbs should be consistent across pages, using same page names, style, etc, should behave as expected to expose a hierarchy and attributes of the page they are on, avoid deviating from the expected behavior
