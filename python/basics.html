Python Basics

Basics

  • Comments: Comments are made by starting a line with #
  • Basic Output: You can output to the console using print("Hello world!"). The argument doesn't have to be a string, it can be a numeric data type as well. You can provide multiple arguments and they will all be concatenated together, you can mix data types with multiple arguments.
  • Basic Input: The input() function will display a provided prompt in the terminal, wait for user input, and then return the input. e.g user_age = input("How old are you? ")
  • Statement terminator: No semicolon or other terminator is used at the end of statements
  • Variables: You can define a variable by simply writing the variable's name (no spaces or special characters except underscore and can't start with a number), followed by the equal sign, followed by it's value. my_variable = "Hello world!". If the variable was previously defined, this will update its value.
  • Variable assignment: += can be used to add a numeric value to a numeric variable, or to concatenate a string value to a string variable, e.g. total_count += 5
  • Basic Numeric Operations: Use +, -, *, /, **, % for addition, subtraction, multiplication, division, exponentiation, modulo (remainder after division)
  • String Concatenation: Use + to concatenate string literals and/or variables together, must convert non-string data to strings to concatenate.

Data Types

  • Strings: String literals can either be in double or single quotes, either is okay just be consistent. "My text string". You can create multi-line strings by enclosing the string in triple quotes, either """, or ''', keeping any line breaks in your code as part of the string.
  • Integer: Or int, is all integers negative, 0 and positive. In modern Python (v3), Ints will be coerced to floats if you perform division on them.
  • Floating-Point Number: Or float, is for decimal numbers
  • Type conversions: Use str(not_a_string) to convert a non-string literal or variable to a string.

Errors

  • SyntaxError: You'll see this in the console if something is wrong with your syntax, along with a copy of the line and a ^ pointing to the problem.
  • NameError: You'll see this in the console if you use a variable name that is not defined.
  • ZeroDivisionError: If you divide by 0.
