Animation

Keyframes

A special element in your CSS that defines the animation

Components

  • Name of the animation
  • Stages of the animation: Animations are broken into stages, where each stage has defined css property values that are reached at a defined percentage of the total animation time
  • CSS Properties: The property values at each stage

Example

        @keyframes bounceIn {
            0% {
                transform: scale(0.1);
                opacity: 0;
            }
            60% {
                transform: scale(1.2);
                opacity: 1;
            }
            100% {
                transform: scale(1);
            }
        }

Animation Properties

Properties assigned to the element to animate

Components

  • animation-name
  • animation-duration: seconds (s) or milliseconds (ms)
  • animation-timing-function: defines how css property values are interpolated between animation stages, e.g., ease (default), linear, ease-in, ease-out, ease-in-out, initial, inherit, custom cubic-bezier curve
  • animation-delay: defined in seconds or milliseconds, specifies a delay before animation begins, or can be a negative value to start instantly start in the middle of the animation
  • animation-iteration-count: specifies number of repeats of the animation, can be a number, or keywords infinite, initial, or inherit
  • animation-direction: can be normal (default), reverse, alternate (normal-reverse-normal-reverse, etc), or alternate-reverse (reverse-normal-reverse-normal etc)
  • animation-fill-mode: defines if the initial and end states of the animation are in effect before/after the animation, as opposed to the element's normal styles, can be normal (default - animation doesn't apply), backwards (initial animation state applied beforehand), forwards (final animation state applied afterwards), both (applies rules of both forwards and backwards)
  • animation-play-state: can be playing or paused, to define if animation should run or stay paused
  • animation: recommended shorthand that combines all of the above properties separated by a space, the first two properties are required, and can combine multiple animations separated by a comma: [animation-name] [animation-duration] [animation-timing-function] [animation-delay] [animation-iteration-count] [animation-direction] [animation-fill-mode] [animation-play-state], [animation-name] [animation-duration] etc.

Example

        div {
            animation-duration: 2s;
            animation-name: bounceIn;
        }
OR 
        div {
            animation: bounceIn 2s;
        }

Prefixes

Probably all outdated but in the past, in addition to unprefixed versions, you needed to include the -webkit- prefixed version for keyframes, animation properties, and transitions for certain browsers. There are libraries that can automate this.

Example

        div {
            -webkit-animation-duration: 2s;
            animation-duration: 2s;
            -webkit-animation-name: bounceIn;
            animation-name: bounceIn;
        }

        @-webkit-keyframes bounceIn { /* styles */ }
        @keyframes bounceIn { /* styles */ }
