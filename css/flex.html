CSS Flexbox

Basics

Primarily used for positioning items in a single direction (whether vertical or horizontal) whereas grid is used for two dimensional layout, although wrapping is used with flex

Flex container: parent element for which display should be set to flex or inline-flex

Flex items: child elements of the flex container

Flex Container Properties

flex-direction: determines major axis orientation

  • row (default) horizontal from left to right
  • row-reverse is from right to left
  • column is vertical from top to bottom
  • column-reverse row is from bottom to top

flex-wrap:

  • nowrap (default prevents items from wrapping and instead following the shrink/grow behavior
  • wrap will wrap rows below current for from top to bottom
  • wrap-reverse will wrap rows above current row - from bottom to top

flex-flow: defines flex-direction and flex-wrap in one property, e.g. flex-flow: column wrap

justify-content: spacing of items along the major axis (default left-right) inside a flex container

  • flex-start
  • flex-end
  • center
  • space-between
  • space-around
  • space-evenly
  • safe/unsafe can use additional safe or unsafe keyword to prevent pushing element out of the container in such a way they can't be scrolled to

align-items: cross axis (default top to bottom) spacing of items inside a flex container, aligns items vertically that are within the same row

  • flex-start
  • flex-end
  • center
  • stretch
  • baseline aligns text baselines in each item
  • safe/unsafe can also use safe or unsafe keyword as with justify-content

align-content: for aligning multiple rows vertically and only applies if flex-wrap is wrap or wrap-reverse, align-items is vertical alignment within each row, and align-content is vertical alignment of the rows themselves

  • flex-start
  • flex-end
  • center
  • stretch
  • space-between
  • space-around
  • safe/unsafe can also use safe or unsafe keyword as with justify-content

Flex Item Properties

Invalid properties: float, clear, and vertical-align have no effect on flex items

order: By default flex items layout in the order in the HTML document, but can define the order explicitly with this, e.g. order: 5 accepts negative numbers too

flex-grow:

  • default is 0 meaning flex items will not grow when parent container is wider than elements
  • if all flex items are 1, then they will fill the full container and each take the same fraction of space
  • can use higher numbers for some elements to take up more of the space than others

flex-shrink:

  • default is 1 meaning flex items automatically shrink when container width is smaller than items

flex-basis: defines the default width of the flex item when the flex-container is exactly wide enough to fit the width of all its flex items without any shrinking or growing, can be px, %, rem, em, etc or use auto to use the element's defined width or height property.

flex: defines flex-grow, flex-shrink, and flex-basis

  • you can also leave off the flex-basis if you only want to declare flex-grow and flex-shrink
  • or you can leave out flex-shrink to define only flex-grow and flex-basis
  • you can't leave out flex-grow to define only flex-shrink and flex-basis
  • flex: 1 (or some other positive interger) means flex: 1 1 0, which basically eliminates consideration of the width of the flex item and simply grows/shrinks it within the flex container to fit whatever fraction is specified (1 in this case)

align-self: allows you to override the default align-items value, same values as align-items
