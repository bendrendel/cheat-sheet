Search Engine Optimization

Basics

  • SEO is the means by which websites appear in search engines.
  • Google holds 85%+ market share. More than 90% of web traffic comes through search engines.
  • "Black Hat" SEO means using deceptive techniques to improve SEO like cramming keywords in hidden places, or purchasing links from reputable sites
  • Black Hat SEO may work temporarily, but search engines will punish the behavior if found out
  • Ultimately good SEO is a long-term game that improves with long-term maintenance

Essentials

  • Title Element: Search Engines inspect your title elements, so create unique titles for each page, avoid vague titles like Home, keep them descriptive but concise, e.g. use "Mary's quick maple bacon donut recipe" instead of "Donut recipe". Don't stuff with keywords or else they could be labelled spam
  • Meta Description: the <meta name="description"> tag summarizes the page's contents that search engines include in the search results. High-quality unique descriptions make your page appear more relevant and can increase traffic. Descriptions should be unique for each page, descriptive but concise, not vague like "home", no keyword stuffing (or could be marked as spam), can contain structured data instead of complete sentences. E.g. Use <meta name="description" content="Mary's simple recipe for maple bacon donuts makes a sticky, sweet treat with just a hint of salt that you'll keep coming back for."> instead of simply content="A donut recipe."
  • Link text: For anchor tags, avoid generic displayed text like 'click here', 'learn more', 'start', etc. Use descriptive text that clearly indicates what kind of content will be found by following the link. This helps users and search engines understand contents and how they relate to each other. Keep text to a few words or short phrase, e.g. use <p>Check out all of our <a href="#">basketball videos</a></p> instead of <p>Check out our basketball videos <a href="#">here</a></p>
  • hreflang: if your site has multiple versions for different languages or regions, then these should be specified in link tags using the hreflang attribute, or it can be done through response headers for your site, or language version info can be added to your sitemap
  • rel="canonical" Use this tag if you have duplicative pages
  • HTTP Error Codes: Make sure pages don't return HTTP error codes (400 and 500 series) they may not index correctly
  • robots.txt: Use a robots.txt file in your project to tell search engine which pages should be crawled and be public. You can also link to a sitemap file to help search engines.
  • Plugins: Don't use browser plugins like Java and Flash. These won't be indexed properly.
  • Meta viewport tag: Always include a <meta name="viewport" content="width=device-width, initial-scale=1"> tag, as search engines will check this often to help rank if your site is mobile friendly.
  • Font Size: Make sure to use font sizes that are at least 12px for 60% or more of your text content. Smaller sizes may require zooming in and are bad for seo
  • Tap Targets: Buttons/links/etc that users tap with a finger on mobile, should be at least 48px x 48px, and at least 8px apart, or else they are difficult to use and will impact seo.

Other Considerations

  • Good Content: Quality content matters above all, consider:
    • Expertise, Authoritativeness, and Trustworthiness (E-A-T)
    • Your Money or Your Life (YMYL) Pages
    • The reputation of websites and creators
    • Mobile user needs
    • Auto-generated content
    • Deceptive page design
    • What low quality pages look like
  • Accessibility: Accessible design tends to overlap with good practices for SEO. Consider:
    • Image alt text
    • Descriptive title and header tags
    • Video transcriptions
    • Content sections
    • Clear, logical sitemaps
    • Colour contrast
  • Javascript:
    • Web-crawlers will give up if a page takes too long to load
    • Try Google's URL Inspection Tool
  • Structured Data:
    • Beyond metadata and semantic markup, there is structured data techniques like Schema
    • This is used by search engines to display rich search results based on the type of content. (e.g. formatted recipe cards in the search results for a recipe)
  • Site Speed:
    • Web crawlers give up on slow sites, and don't prioritize them for users
    • Avoid big images (multiple megabytes), link to big images if you must.
    • Streamline code, removing superfluous HTML, CSS, Javascript
    • PageSpeed Insights will help you improve site speed
  • Mobile First:
    • Google now indexes based on the mobile version of your site, not the desktop version.
