Operating Systems

Basics

  • Operating systems provide an interface between the computer user and hardware. It manages the hardware and software resources of a system (e.g. processor, disk space, memory, keypad, network connection, etc), and provides a stable and consistent way for other software to interact with the hardware without knowing hardware details. The first task involves managing CPU usage, memory, i/o, etc. to ensure demands on limited system resources are used for the greatest good of the user and applications. The second task involves providing a consistent API to software developers that manages all the hardware details so other software doesn't have to and will still run on different machines, when hardware is upgraded, if memory capacity changes, regardless of what type of printer is connected, etc. Users interact with the OS and its managers through the user interface. Windows, Mac OS, Linux, and Unix are the major Operating Systems. There are four essential managers of every operating system that work together:
    • Memory Manager: Manages RAM (main or primary memory - a large array of addressed bytes or words that can be accessed directly by the CPU and is where programs are loaded to to run)
      • Keeps track of what memory is being used for what purpose, what memory is not in use
      • Decides which processes get memory and how much
      • Allocates and deallocates memory when a program requests to do so or if determined it can be deallocated
    • Process Manager: Manages which process gets to use the CPU (central processing unit) and for how long (process scheduling)
      • The traffic controller keeps track of the status of processes and the processor
      • Allocates and deallocates the processor to processes
    • Device Manager: Manages device communication via their respective drivers
      • The I/O controller keeps track of all devices
      • Decides which process gets each device, when and for how long
      • Allocates and deallocates devices
    • File Manager: Manages the systems stored resources, usually organized in a heirarchical directory structure
      • The File System keeps track of information, locations, uses, status, of all files
      • Decides who gets the resources (including user permissions)
      • Allocates and de-allocates the resources
    • Other Important OS Activiites
      • Network Manager: Allows sharing of hardware and software resources while controlling access
      • Security: Password/user management
      • System Performance: monitors delay between requests and responses of system
      • Job Accounting: Keeps track of time and resources used by jobs and users
      • Error Detecting: Produces dumps, traces, error messages, and other aides
      • Coordination between other users and software: Assignment of compilers, interpreters, and assemblers to users
