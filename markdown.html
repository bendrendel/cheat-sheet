Overview

  • Markdown is a text-formatting language designed for the web with PERL
  • Widely used in software/technical documentation, many communication applications like skype, major websites like wordpress, writing blogs posts for a website, to take notes, implemented in emails, and multiple places in GitHub (Gists, comments in issues and pull requests, and files with markdown extension)
  • It is designed to be lighter weight than HTML to be more accessible, but can't do nearly as much as HTML
  • Markdown processors or parsers are included in certain software that can read markdown, it converts markdown to HTML
  • File extension is .markdown or .md

Markdown Syntax

Headers

  • # This will make an h1 tag
  • ###### You can go up to an h6 tag with multiple # signs

Emphasis

  • *Italic text enclosed in one asterisk (or one underscore)*
  • **Bold text eclosed in two asterisks (or two underscores)**
  • _Example of combining italics with **bold** inside_

Lists

  • * Bulleted lists begin each item with an asterisk (or a dash)
      * Create a sub-bullet with two spaces and then an asterisk (or a dash)
  • 1. Create numbered lists by simply starting line with number and period
      1a. Here is sub-item for a numbered list

Links

  • [Displayed link name](url)
  • url could be local or on the web like https://google.com

Images

  • ![Alt text](url)
  • url could be local like ./images/logo.png or on the web

Blockquotes

  • > Start a line with an arrow bracket to create a formatted block quote
  • > Can include multiple lines in the blockquote

Inline Code

  • For code snippets (html, css, js, anything) use backticks `const myVar` for special formatting

Task Lists

  • In descriptions and comments of Issues and Pull Requests, and Gist coments and Gist markdown files, you can include a task list
  • - [x] Task lists include a checkbox, this item is checked
  • - [ ] This item will be unchecked
  • - [ ] Can also use ordered lists
  • - [ ] If included in the first comment of an issue, github will show a progress bar in the list of issues

Tables

  • The below markup will be formatted as a table using | for col breaks, and -- for header break
  • First Header | Second Header
  • ------------ | -------------
  • Content in row 1 col 1 | Content in row 1 col 2
  • Content in row 2 col 1 | Content in row 2 col 2

Links

  • Any URL in the text like http://www.github.com/ will automatically be a clickable link
  • In description and comments of Issues and Pull Requests, SHA references like below automatically become a link to the commit on Github
    • 16c999e8c71134401a78d4d46435517b2271d6ac
    • bendrendel@16c999e8c71134401a78d4d46435517b2271d6ac
    • bendrendel/repository-name@16c999e8c71134401a78d4d46435517b2271d6ac
  • In description and comments of Issues and Pull Requests, any number referring to an Issue or Pull Request like below becomes a link
    • #1
    • bendrendel#1
    • bendrendel/repository-name#1

Username @ mentions

  • In description and comments of Issues and Pull Requests, Using @bendrendel or other username will notify the person or team to view the comment
