Starting a React App

Create React App CLI: create-react-app is a node package you can install globally that provides CLI commands to create a boilerplate react app

  • npm install -g create-react-app Run this command to install create-react-app on your computer globally (-g), this only needs to be done once, not for each project. As of node 5.2, this is no longer recommended, and instead you should use npx a package runner included with node
  • npx create-react-app {app-name} Run this command to create a new react project in the current working directory, the app name should be all lowercase, a new directory for the project will be made with the provided name and all the react boilerplate directories, files, and developer settings
  • npm install react-scripts@latest use this in your project to install the latest version of create react app

React App Directory Structure: create-react-app will create the following structure

  • .gitignore create-react-app does not start a git repo for you, but it does create a .gitignore file with standard things to ignore when you do start a git repo
  • package.json it will create the package.json file with dependencies, scripts, and basic metadata for you
  • package-lock.json this file contains the exact dependency tree in /node_modules along with all versions, helpful for teams to ensure they have same versions of everything, also contains history of changes to package.json to inspect dependency changes
  • README.md will make a starter readme file
  • /node_modules it will create the node_modules directory with react, react-dom, and react-scripts installed and subdependencies
  • /public this directory is for assets to serve directly without additional processing by webpack, will include index.html (entry point for web app), favicon.ico (header icon), manifest.json (specifies behavior if added to Android user's home screen), logo192.png, logo512.png, and robots.txt
  • /src this directory is for javascript to be processed by webpack, it has the main app js, style, and test files (App.js, App.css, and App.test.js). it also has the index.js and associated index.css that provide an entry to the app and kick off the registerServiceWorker.js, which caches and updates files for end-users to allow offline viewing and faster loading. Also includes logo.svg, serviceWorker.js, and setupTests.js
    • /Components this directory actually is NOT created by create-react-app, but it is common to add it under src
      • /ComponentName each component its own folder with the capitalized name of the component, e.g. ./src/components/App/ or ./src/components/MyComponent/ Commonly you will start with a main component called App and you cab move the default App.js and App.css in the src folder into the Components/App folder. Be sure to update the App.js import statement in index.js to the new location of App.js
        • /ComponentName.js the component will have a .js, use import './ComponentName.css at the top of the file immediately after the react import to import the css file so the styles are applied
        • /ComponentName.css the css file styles the jsx returned by the component and simply refers to the elements, ids, classes etc in the jsx, looks like a regular css file, no special export statement or anything
        • Assets like images used in the components css or jsx should be stored in this folder as well
    • /views this directory also is NOT created by create-react-app but is common to add it under src to organize views

Common Setup Tasks

npm start once set up, run this command in the react app's directory to begin serving a local development server at http://localhost:3000/, changes to the app's files will live-update at the served site

favicon delete the default favicon found in the ./public/ directory and replace it with your own, file should be named favicon.ico

css reset add a css reset file to the ./public/ directory and link to it in ./public/index.html, to apply the reset across the entire app

fonts add links to fonts like google fonts in the public/index.html file to use throughout app

create components directory create the components directory and create the App component folder first, move the default App.js/App.css/App.test.js files into it (can delete App.test.js if not using), and then update the link to src/App/App.js in src/index.js (index.js and index.css should stay where they are directly under ./src/)

React Developer Tools

React Developer Tools Chrome extension developed by facebook to help inspect React components

  • Icon Icon of React Developer Tools is grayed out on pages that are not React, orange if site is a React App in development build, black if page is using production build of React
  • Components tab Adds a new components tab to chrome dev tools, allows you to inspect components, check setting for components tab to make sure no filters are applied that is hiding components, click on a component to see detail and then you can directly edit props, state, and hooks and also access it in the console tab using $r, with react dev tools note you can't add a state to a component that doesn't have one, the component must be initialized with state in the actual code
  • Profiler tab Also a new tab added to chrome dev tools for inspecting performance information about the react app
